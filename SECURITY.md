# Security

## Reporting an issue

If you discover a security issue in AttieLab-QTL-Viewer, please report it
**privately by email** rather than opening a public GitHub issue:

- Charles Opara — copara@wisc.edu

Please include:

- A description of the issue and the affected component (data layer, S3
  configuration, Shiny session handling, etc.).
- Steps to reproduce, or a minimal example demonstrating the problem.
- Any logs or error messages that help diagnose the issue.

We will acknowledge your report within a few business days and follow up
with a remediation plan.

## Supported versions

Only the latest released version is actively maintained. Please update to
the most recent release before reporting issues.

## Scope

This repository contains the application source code. Reports about the
deployed instance (the live web application URL) should also be sent
privately to the address above so the deployment can be patched promptly.

## Out of scope

- Vulnerabilities in upstream R packages (please report those to the
  package maintainers directly).
- Issues in the Posit Connect platform itself (report to your institution's
  Connect administrator).
